SQLServer安装向导

安装前建议关闭所有Windows 应用程序，包括Windows 资源管理器，建议关闭防火墙

插入SQL Server 2019 安装包。右键“setup.exe”，选择“以管理员身份运行”，运行过程如果出现弹窗（是否允许更改），选择“是”

在SQL Server 安装中心的左侧窗格中单击“安装”，在SQL Server 安装中心的右侧窗格中选择“新的SQL Server 独立安装或添加功能到现有安装”

安装检查

安装检查

无需输入产品序列号，直接下一步

5.阅读许可并勾选“我接受许可条款和隐私声明”并点击“下一步”

6.点击下一步

安装规则检查，如下图，检查通过，点击“下一步”（由于未关闭防火墙产生警告）
注意：若安装规则检查出现错误，单击“显示细节”，检查安装失败的组件或提示警告；点击“取消”停止安装，并解决对应问题提示

功能选择

选择安装功能：
a. 在实例功能下，选择数据库引擎服务
b. 在共享功能下，选择：
• 客户端工具连接
• Integration Services
• 客户端工具后向兼容
• SQL Server 联机丛书
• 管理工具– 基本
• 管理工具– 完整

c.安装路径建议默认安装,点击下一步

实例配置

首次安装选择默认安装（可多次安装配置不同实例，建议为PDM、Electrical、Manage等不同软件分配不同数据库实例），点击“下一步”
注意：若计算机/服务器之前已安装SQL Server，则需要自定义不同实例名称

数据库配置

  1. 数据库引擎配置
    a.选择混合模式（SQL Server 身份验证和Windows 身份验证）身份验证。SOLIDWORKS PDM
    使用SQL Server 身份验证进行数据库通信。一般不要改变数据目录或文件流选项卡中的设置。
    b.为系统管理员帐户(sa) 输入安全级别较高的密码。不要将密码保留为空。有关密码规则， 请单击帮助。重新输入密码进行确认。记住该密码，因为在设置 SOLIDWORKS PDM 文件库数据库时要用到。
    c.数据库引擎配置/数据目录选项卡
    d.若需要设置创建的数据库的默认存储文件夹，更选择【数据目录】界面，更改用户数据库目录和用户数据库日志目录的位置

  2. 准备安装，点击“安装”

安装成功，点击“关闭”如图界面，关闭“SQL Server安装中心”
